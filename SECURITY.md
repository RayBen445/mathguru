# Security Policy

## Supported Versions

The latest release on `main` is supported for security updates.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Instead:

1. Open a private security advisory on GitHub, or
2. Contact maintainers through private channels listed in repository metadata.

Include:

- affected version
- reproduction steps
- impact assessment
- suggested mitigation (if available)

## Security Practices

- CI includes `npm audit` checks.
- Releases are validated before npm publishing.
- Dependency updates should be tested and reviewed.
